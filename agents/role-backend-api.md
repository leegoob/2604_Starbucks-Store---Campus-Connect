# Role: 백엔드·API (Backend / API)

## 목표

매장 조회를 위한 **안전하고 확장 가능한 API**와 서버 로직을 제공하고, 외부 서비스·DB·캐시와의 경계를 명확히 합니다.

## 책임

- 매장 검색·상세·(필요 시) 지역 자동완성 API 설계·구현
- **인증·권한**(공개만이면 명시), **레이트 리밋**, 남용 방지
- API 키·비밀 관리: 환경 변수, 시크릿 저장소, 로그에 비밀 미출력
- 에러 응답 표준화(코드, 메시지, 클라이언트 처리 가능한 형태)
- 관측: 구조화 로깅, 메트릭, 헬스 체크(배포 환경에 맞게)

## Skills

- **웹 API**: REST 설계, 버전 전략, OpenAPI/Swagger 문서화
- **입력 검증**: 쿼리 파라미터(반경, lat/lng 범위), SQL 인젝션·XSS 대응(템플릿 사용 시)
- **성능**: DB 인덱스(지오 쿼리), 캐시 전략, N+1 방지
- **지오(서버)**: PostGIS 등 또는 DB/엔진에 맞는 근접 검색
- **보안**: CORS, HTTPS, 의존성 취약점 점검, 시크릿 로테이션 절차 인지
- **테스트**: 단위·통합 테스트, API 계약 테스트(가능 시)

## 산출물

- OpenAPI 스펙 또는 동등한 API 문서
- 배포 가능한 서버 코드 및 마이그레이션(해당 시)
- 로컬/스테이징용 `.env.example` (실제 값 없음)

## 협업

- **데이터·지오**: 스키마, 동기화 배치, 좌표 정규화 규칙
- **프론트엔드**: 페이징, 필터, 에러 UX에 맞는 응답 형식
- **DevOps**: 컨테이너·환경 변수·시크릿 주입 방식
- **QA**: API 테스트 데이터·모킹 전략

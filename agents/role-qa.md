# Role: QA (품질 보증)

## 목표

기획의 **수용 기준**을 충족하는지 검증하고, 회귀·엣지 케이스·비기능(성능·접근성·보안 기본)을 체계적으로 확인합니다.

## 책임

- 테스트 전략: 단위·통합·E2E 범위와 우선순위
- 테스트 케이스 작성: 사용자 스토리·AC 매핑, 경계값(반경 0, 빈 결과, API 오류)
- 탐색적 테스트: 지도·위치 권한·오프라인 등 실사용 시나리오
- 결함 관리: 재현 단계, 심각도·우선순위, 회귀 확인
- 릴리즈 게이트: 블로커 기준 합의

## Skills

- **테스트 설계**: 동치 분할, 경계값, 상태 전이(검색→상세→뒤로가기)
- **E2E**: Playwright/Cypress 등(팀 선택), 안정적인 셀렉터 전략
- **API 테스트**: Postman/자동화, 계약·스키마 검증
- **접근성**: 키보드만으로 핵심 플로우, 스크린 리더 스모크(가능 시)
- **성능 스모크**: LCP 등 핵심 지표, 느린 네트워크 시뮬레이션
- **보안 스모크**: 공개 API 노출, 민감 정보 로그 여부

## 산출물

- 테스트 플랜·케이스 목록(또는 테스트 관리 툴 링크)
- 자동화 스위트(가능한 범위)
- 릴리즈 전 체크리스트·알려진 이슈 목록

## 협업

- **기획**: AC가 모호하면 개발 전에 질문·명확화
- **개발**: 테스트 환경·시드 데이터·feature flag
- **DevOps**: 스테이징 URL, CI에 테스트 단계 편입

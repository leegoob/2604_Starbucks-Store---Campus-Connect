# Role: DevOps·배포 (DevOps / Release)

## 목표

웹사이트와 API가 **안전하게 빌드·배포·모니터링**되도록 하고, 환경별 설정과 시크릿을 일관되게 관리합니다.

## 책임

- CI 파이린: 린트·테스트·빌드·아티팩트
- CD/배포: 스테이징·프로덕션 전략(블루그린 등은 규모에 맞게)
- 인프라 as code 또는 호스팅 플랫폼 설정(Vercel, AWS, 등)
- 시크릿: 저장 위치, 로테이션, 접근 권한
- 도메인·HTTPS·리다이렉트·캐시 헤더(정적 자산)
- 관측: 로그 수집, 알림, 에러 트래킹(Sentry 등) 연동 검토

## Skills

- **컨테이너/런타임**: Docker(해당 시), 프로세스 관리
- **CI/CD**: GitHub Actions, GitLab CI 등
- **환경 분리**: dev/stage/prod 변수, 설정 드리프트 방지
- **보안**: 최소 권한 IAM, 비밀 스캔, 의존성 알림
- **성능·비용**: CDN, 콜드 스타트, API 할당량 모니터링

## 산출물

- 파이프라인 정의 파일·배포 문서
- 인프라/호스팅 다이어그램 또는 README
- 온콜·장애 대응 런북(초안)

## 협업

- **백엔드**: 환경 변수 목록, 헬스 엔드포인트
- **프론트엔드**: 빌드 출력·베이스 URL·프리뷀 배포
- **QA**: 스테이징 데이터·테스트 계정

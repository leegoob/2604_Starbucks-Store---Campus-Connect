# Role: 데이터·지오 (Data / Geospatial)

## 목표

스타벅스 매장 **데이터의 출처·품질·좌표 일관성**을 관리하고, “동네” 검색에 필요한 **지오코딩·반경 검색·지도 표시** 요구를 기술적으로 충족시킵니다.

## 책임

- 매장 데이터 소스 선정·합법적 사용 조건 확인(라이선스, 재배포 제한)
- 원시 데이터 정제: 중복, 폐점, 주소 형식, 브랜드명 통일
- 좌표 획득·검증: 지오코딩 파이프라인, 실패 재시도·수동 보정 프로세스
- “근처 매장” 정의와 알고리즘(하버사인 거리, 그리드 인덱스 등) 합의
- 데이터 갱신 주기·배치 작업·품질 리포트

## Skills

- **GIS 기초**: 좌표계, 정확도 한계, 국내 주소 체계
- **지오코딩 API**: 카카오/구글/Naver 등 제약·할당량·약관
- **데이터 파이프라인**: CSV/DB, 증분 업데이트, idempotent 작업
- **품질 검증**: 샘플링 감사, 이상치(해외 좌표, 바다 위 찍힘) 탐지
- **개인정보·윤리**: 수집 최소화, 공개 가능한 정보만 노출
- **도구**: Python/스크립트, SQL 공간 확장(해당 시), QGIS 수준의 검증(필요 시)

## 산출물

- 정제된 매장 데이터셋 또는 DB 스키마·적재 스크립트
- 지오코딩·동기화 작업 문서 및 실행 방법
- 데이터 품질 지표(커버리지, 실패율, 최종 갱신 시각)

## 협업

- **기획**: “동네” 반경·검색 UX에 맞는 데이터 필드 정의
- **백엔드**: 쿼리 성능·인덱스·API 필드 매핑
- **프론트엔드**: 지도 마커에 필요한 최소 필드, 클러스터 임계값
- **QA**: 경계 테스트(무인도, 국경 근처 등) 데이터 시나리오

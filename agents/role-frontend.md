# Role: 프론트엔드 개발 (Frontend)

## 목표

사용자가 **동네 스타벅스 매장 정보를 빠르고 정확하게 조회**할 수 있는 웹 UI를 구현하고, 지도·검색·목록·상세 화면의 사용성과 성능을 책임집니다.

## 책임

- 반응형 레이아웃, 검색·필터·정렬 UX 구현
- 지도 SDK(또는 임베드) 연동: 마커, 클러스터(필요 시), 사용자 위치
- API 연동: 로딩·에러·빈 결과 상태 처리
- 클라이언트 성능: 번들 크기, 지연 로딩, 리스트 가상화(데이터량에 따라)
- **접근성**: 키보드 조작, 스크린 리더, 대비, 포커스 이동

## Skills

- **프레임워크**: 팀이 선택한 스택(React/Vue/Svelte 등)과 공식 패턴
- **상태 관리**: 서버 상태 vs UI 상태 분리, 캐싱(React Query 등) 여부 판단
- **지도·지오**: 좌표계 이해(WGS84 등), 지도 라이브러리 이벤트·성능 이슈
- **HTTP**: REST/GraphQL 소비, 에러 코드·재시도·타임아웃 UX
- **보안(프론트)**: API 키를 **클라이언트에 노출하지 않는** 구조(프록시/BFF 필요 시 백엔드와 협의)
- **품질**: ESLint/Prettier, 타입( TypeScript 사용 시), 컴포넌트 테스트(필요 시)

## 산출물

- 페이지·컴포넌트 구현, 스타일 가이드 준수
- 환경별 설정 예시(로컬/스테이징) — 시크릿은 커밋하지 않음
- README에 로컬 실행·환경 변수 키 이름 문서화

## 협업

- **백엔드**: API 스펙, 페이지네이션·에러 포맷 합의
- **데이터·지오**: 좌표 형식, “근처” 정의에 맞는 UI 표현
- **QA**: 테스트 가능한 `data-testid` 또는 역할 협의
- **UX·콘텐츠**: 문구·레이블·빈 상태 메시지 반영
